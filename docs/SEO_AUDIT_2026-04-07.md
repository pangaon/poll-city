## 3. Sitemap Coverage (`src/app/sitemap.ts`)

**Covered:**
- `/` (home), `/pricing`, `/officials`, `/social`, `/social/officials`, `/social/polls`
- `/resources`, `/help`, `/how-polling-works`, `/verify-vote`, `/terms`, `/privacy-policy`, `/login`
- Dynamic: `/candidates/[slug]` (from DB)
- Dynamic: `/social/officials/[id]` (from DB)

**Missing from sitemap:**
- `/social/polls/[id]` — individual poll pages are not included
- `/analytics` — if public, should be listed

## 4. OG Images

- **No dedicated OG image files exist.** No `opengraph-image.tsx`, `opengraph-image.png`, or `twitter-image` files anywhere in the app.
- Root layout sets `openGraph.images` and `twitter.images` to `/logo.png` — this is a fallback logo, not a purpose-built 1200x630 OG image.
- Candidate pages and official pages have no per-page OG images.

## 5. JSON-LD Structured Data

- Root layout (`src/app/layout.tsx`) has a `SoftwareApplication` JSON-LD block. This is correct and well-formed.
- **No additional structured data on other pages:**
  - Candidate pages should have `Person` or `PoliticalCandidate` schema
  - Official detail pages should have `Person` + `GovernmentOrganization` schema
  - Poll pages could use `Question`/`Survey` schema

## 6. Recommendations

### High Priority
1. **Add `generateMetadata` to `src/app/social/officials/[id]/page.tsx`** — These are crawlable, indexed pages. Each official should have a unique title (e.g., "Jane Smith, Ward 5 Councillor | Poll City") and description.
2. **Add `generateMetadata` to `src/app/social/polls/[id]/page.tsx`** — Each poll should have a unique title and description for social sharing.
3. **Create a proper OG image** — Design a 1200x630 PNG or use Next.js `opengraph-image.tsx` (ImageResponse API) to generate branded OG images. At minimum, place a static `opengraph-image.png` in `src/app/`.
4. **Add `/social/polls/[id]` to sitemap** — Dynamic poll pages are missing from the sitemap.

### Medium Priority
5. **Add page-level metadata to `src/app/social/officials/page.tsx`** — Title: "Find Your Representatives | Poll City Social".
6. **Add page-level metadata to `src/app/social/polls/page.tsx`** — Title: "Live Polls | Poll City Social".
7. **Add `Person` JSON-LD to candidate and official detail pages** for richer search results.

### Low Priority
8. **Add page-level metadata to `src/app/social/page.tsx`** — Even though the layout provides a default, a page-specific title improves click-through.
9. **Consider dynamic OG images for candidate/official pages** using Next.js `ImageResponse` for social media previews with candidate name and photo.
10. **Add `robots.txt`** if not already present (not checked in this audit) to complement the sitemap.
