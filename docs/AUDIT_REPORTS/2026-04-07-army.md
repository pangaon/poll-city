## 3. Uses of `any` Type in API Routes

**Count:** 90+ occurrences across 30+ files

### Most Common Patterns
1. **`(session.user as any).activeCampaignId`** -- ~15 occurrences. Indicates `Session` type is not extended for the custom user shape. Should be fixed globally with a `next-auth.d.ts` declaration.
2. **`supportLevel as any` / `as any[]`** -- ~30 occurrences. Prisma enum values cast to `any` because string literals don't match the generated enum type. Fix: import and use the Prisma enum.
3. **`type: "phone_call" as any`** -- ~5 occurrences. Same enum mismatch.
4. **`catch (err: any)`** -- ~5 occurrences. Should use `unknown` and narrow.
5. **`baseWhere: any`** -- 3 occurrences (volunteers, signs, donations). Should use `Prisma.ContactWhereInput`.
6. **`require("twilio") as any`** -- 2 occurrences. Dynamic import without types.

### Highest Risk `any` Usages
- `api/admin/fix-slugs` -- `slug: null as any` in a where clause
- `api/import/route.ts` -- `(existing as any).federalDistrict` bypasses type safety on merge logic
- `api/stripe/webhook` -- `(invoice as any).subscription` may fail silently if Stripe types change
